渐进式Web应用（PWA）开发指南

渐进式Web应用（Progressive Web App，简称PWA）是一种结合了Web和原生应用优势的新型应用形式。它既能像Web应用一样通过URL访问，无需安装，又能像原生应用一样提供离线访问、推送通知、桌面图标等功能。本文将深入探讨PWA的核心概念、技术实现和最佳实践，帮助你构建现代化的Web应用体验。

PWA的价值：根据Google的研究，实施PWA的网站平均可以看到52%的转化率提升，4倍的页面浏览量增长，以及53%的跳出率下降。PWA正在成为现代Web开发的标准实践。

一、PWA的核心特性

PWA具有以下核心特性，使其区别于传统的Web应用：

离线工作能力

通过Service Worker缓存资源，即使在网络不可用的情况下也能提供基本功能。

可安装

用户可以将应用添加到主屏幕，无需通过应用商店下载安装。

推送通知

支持发送推送通知，提高用户参与度和留存率。

性能优化

快速加载、流畅的交互体验，即使在网络条件较差的环境下。

1. 渐进增强

PWA的核心思想是渐进增强（Progressive Enhancement）。它从一个基础的Web应用开始，然后根据浏览器支持的特性逐步添加增强功能。这样可以确保在所有浏览器中都能提供基本功能，同时在支持的浏览器中提供更高级的体验。

2. 响应式设计

PWA应该在任何设备上都能正常工作，从手机到桌面电脑。这就要求应用使用响应式设计原则，适应不同的屏幕尺寸和输入方式。

二、PWA的技术基础

PWA基于三个核心技术构建：Service Worker、Web App Manifest和Push API。让我们逐一了解这些技术。

1. Service Worker

Service Worker是一个运行在浏览器后台的脚本，独立于网页，能够拦截和修改导航和资源请求，以及管理缓存资源。它是实现离线功能的关键技术。

注意：Service Worker只能在HTTPS环境下运行，出于安全考虑。开发环境下（localhost）可以在HTTP环境下使用。

注册Service Worker

在你的应用JavaScript中，首先需要注册Service Worker：

// 检查浏览器是否支持Service Worker
if ('serviceWorker' in navigator) {
    window.addEventListener('load', () => {
        navigator.serviceWorker.register('/service-worker.js')
            .then(registration => {
                console.log('Service Worker 注册成功:', registration.scope);
            })
            .catch(error => {
                console.log('Service Worker 注册失败:', error);
            });
    });
}

Service Worker生命周期

Service Worker有以下几个生命周期事件：

  • install：当Service Worker首次被注册时触发，通常用于预缓存资源。
  • activate：当Service Worker激活时触发，通常用于清理旧的缓存。
  • fetch：当发生网络请求时触发，可以拦截请求并返回缓存的资源。
  • push：当收到推送通知时触发。
  • sync：当网络恢复时触发，用于同步后台数据。

基本的Service Worker实现

下面是一个基本的Service Worker实现，包括资源缓存和离线访问功能：

// service-worker.js

// 缓存名称，用于版本控制
const CACHE_NAME = 'my-pwa-cache-v1';

// 需要缓存的资源列表
const urlsToCache = [
    '/',
    '/index.html',
    '/css/style.css',
    '/js/main.js',
    '/images/logo.svg',
    '/offline.html'
];

// Install事件 - 预缓存资源
self.addEventListener('install', event => {
    event.waitUntil(
        caches.open(CACHE_NAME)
            .then(cache => {
                console.log('打开缓存');
                return cache.addAll(urlsToCache);
            })
            .then(() => self.skipWaiting()) // 立即激活新的Service Worker
    );
});

// Activate事件 - 清理旧缓存
self.addEventListener('activate', event => {
    const cacheWhitelist = [CACHE_NAME];
    event.waitUntil(
        caches.keys().then(cacheNames => {
            return Promise.all(
                cacheNames.map(cacheName => {
                    if (cacheWhitelist.indexOf(cacheName) === -1) {
                        return caches.delete(cacheName);
                    }
                })
            );
        }).then(() => self.clients.claim()) // 控制未被控制的客户端
    );
});

// Fetch事件 - 拦截请求并提供缓存资源
self.addEventListener('fetch', event => {
    event.respondWith(
        caches.match(event.request)
            .then(response => {
                // 如果在缓存中找到资源，直接返回
                if (response) {
                    return response;
                }
                
                // 否则尝试从网络获取
                return fetch(event.request)
                    .then(response => {
                        // 检查响应是否有效
                        if (!response || response.status !== 200 || response.type !== 'basic') {
                            return response;
                        }
                        
                        // 克隆响应，一份用于缓存，一份返回给浏览器
                        const responseToCache = response.clone();
                        
                        caches.open(CACHE_NAME)
                            .then(cache => {
                                cache.put(event.request, responseToCache);
                            });
                        
                        return response;
                    })
                    .catch(() => {
                        // 网络请求失败时，如果请求的是HTML页面，返回离线页面
                        if (event.request.headers.get('accept').includes('text/html')) {
                            return caches.match('/offline.html');
                        }
                    });
            })
    );
});

// Push事件 - 处理推送通知
self.addEventListener('push', event => {
    if (!event.data) return;
    
    try {
        const data = event.data.json();
        const options = {
            body: data.body,
            icon: '/images/icon-192x192.png',
            badge: '/images/badge-72x72.png',
            data: {
                url: data.url
            }
        };
        
        event.waitUntil(
            self.registration.showNotification(data.title, options)
        );
    } catch (e) {
        console.error('Push事件处理失败:', e);
    }
});

// Notificationclick事件 - 处理通知点击
self.addEventListener('notificationclick', event => {
    event.notification.close();
    
    if (event.notification.data && event.notification.data.url) {
        event.waitUntil(
            clients.openWindow(event.notification.data.url)
        );
    }
});

3. Push API

Push API允许服务器向客户端发送消息，即使客户端没有打开应用。结合Service Worker，我们可以实现推送通知功能。

请求通知权限并订阅推送

// 请求通知权限并订阅推送
function registerForPushNotifications() {
    // 检查浏览器是否支持推送API
    if ('Notification' in window && 'PushManager' in window) {
        Notification.requestPermission().then(permission => {
            if (permission === 'granted') {
                console.log('通知权限已授予');
                return navigator.serviceWorker.ready;
            } else {
                console.log('通知权限被拒绝');
            }
        }).then(registration => {
            if (!registration) return;
            
            // 向你的服务器获取公钥
            return fetch('/api/vapid-public-key')
                .then(response => response.text())
                .then(publicKey => {
                    // 将base64字符串转换为Uint8Array
                    const convertedPublicKey = urlBase64ToUint8Array(publicKey);
                    
                    // 订阅推送
                    return registration.pushManager.subscribe({
                        userVisibleOnly: true,
                        applicationServerKey: convertedPublicKey
                    });
                });
        }).then(subscription => {
            if (!subscription) return;
            
            // 将订阅信息发送到服务器
            return fetch('/api/subscribe', {
                method: 'POST',
                headers: {
                    'Content-Type': 'application/json'
                },
                body: JSON.stringify(subscription)
            });
        }).then(response => {
            if (response.ok) {
                console.log('推送订阅成功');
            }
        }).catch(error => {
            console.error('推送订阅失败:', error);
        });
    }
}

// 辅助函数：将base64字符串转换为Uint8Array
function urlBase64ToUint8Array(base64String) {
    const padding = '='.repeat((4 - base64String.length % 4) % 4);
    const base64 = (base64String + padding)
        .replace(/-/g, '+')
        .replace(/_/g, '/');
    
    const rawData = window.atob(base64);
    const outputArray = new Uint8Array(rawData.length);
    
    for (let i = 0; i < rawData.length; ++i) {
        outputArray[i] = rawData.charCodeAt(i);
    }
    return outputArray;
}

// 在用户交互时调用，例如点击按钮
button.addEventListener('click', registerForPushNotifications);
安全注意事项：推送通知需要使用VAPID（Voluntary Application Server Identification）密钥对来确保只有你的服务器才能发送通知。在生产环境中，请确保安全存储和使用这些密钥。

三、PWA的缓存策略

选择正确的缓存策略对于PWA的性能和用户体验至关重要。以下是几种常用的缓存策略：

1. 缓存优先（Cache First）

先尝试从缓存中获取资源，如果缓存中没有，再从网络获取并更新缓存。适用于不经常变化的静态资源。

self.addEventListener('fetch', event => {
    event.respondWith(
        caches.match(event.request)
            .then(response => {
                // 如果缓存中有响应，直接返回
                if (response) {
                    return response;
                }
                
                // 否则从网络获取
                return fetch(event.request)
                    .then(response => {
                        // 检查响应是否有效
                        if (!response || response.status !== 200 || response.type !== 'basic') {
                            return response;
                        }
                        
                        // 克隆响应并缓存
                        const responseToCache = response.clone();
                        caches.open(CACHE_NAME)
                            .then(cache => {
                                cache.put(event.request, responseToCache);
                            });
                        
                        return response;
                    });
            })
    );
});

2. 网络优先（Network First）

先尝试从网络获取资源，如果网络请求失败，再从缓存中获取。适用于经常更新的内容。

self.addEventListener('fetch', event => {
    event.respondWith(
        fetch(event.request)
            .then(response => {
                // 克隆响应并缓存
                const responseToCache = response.clone();
                caches.open(CACHE_NAME)
                    .then(cache => {
                        cache.put(event.request, responseToCache);
                    });
                
                return response;
            })
            .catch(() => {
                // 网络请求失败时，从缓存中获取
                return caches.match(event.request);
            })
    );
});

3. 快速回退（Stale While Revalidate）

先返回缓存的资源（如果有），同时在后台从网络获取最新版本并更新缓存。这是一种平衡性能和新鲜度的策略。

self.addEventListener('fetch', event => {
    event.respondWith(
        caches.match(event.request)
            .then(cachedResponse => {
                // 无论缓存中是否有响应，都从网络获取最新版本
                const fetchPromise = fetch(event.request)
                    .then(networkResponse => {
                        // 克隆响应并更新缓存
                        const responseToCache = networkResponse.clone();
                        caches.open(CACHE_NAME)
                            .then(cache => {
                                cache.put(event.request, responseToCache);
                            });
                        
                        return networkResponse;
                    })
                    .catch(() => {
                        // 如果缓存中有响应但网络请求失败，返回缓存的响应
                        if (cachedResponse) {
                            return cachedResponse;
                        }
                        // 如果缓存中也没有响应，返回一个默认响应或离线页面
                    });
                
                // 如果缓存中有响应，立即返回，否则等待网络请求
                return cachedResponse || fetchPromise;
            })
    );
});

4. 资源特定策略

针对不同类型的资源使用不同的缓存策略：

self.addEventListener('fetch', event => {
    const url = new URL(event.request.url);
    
    // 对不同类型的资源使用不同的缓存策略
    if (url.pathname.startsWith('/api/')) {
        // API请求使用网络优先策略
        event.respondWith(networkFirst(event.request));
    } else if (url.pathname.match(/\.(?:js|css|html)$/)) {
        // JavaScript、CSS和HTML文件使用快速回退策略
        event.respondWith(staleWhileRevalidate(event.request));
    } else if (url.pathname.match(/\.(?:png|jpg|jpeg|svg|gif)$/)) {
        // 图片使用缓存优先策略
        event.respondWith(cacheFirst(event.request));
    }
});

四、PWA的安装体验

PWA可以被添加到用户的主屏幕，提供接近原生应用的体验。浏览器通常会自动提示用户安装，但我们也可以提供自定义的安装按钮。

1. 检测安装事件

浏览器会在满足一定条件时触发beforeinstallprompt事件，我们可以捕获这个事件并显示自定义的安装提示：

// 存储安装事件，以便稍后触发
let deferredPrompt;

// 监听beforeinstallprompt事件
window.addEventListener('beforeinstallprompt', (e) => {
    // 阻止Chrome 67及更早版本自动显示安装提示
    e.preventDefault();
    // 存储事件以便稍后触发
    deferredPrompt = e;
    // 显示自定义安装按钮
    showInstallPromotion();
});

// 自定义安装按钮点击事件
installButton.addEventListener('click', async () => {
    if (!deferredPrompt) {
        return;
    }
    
    // 显示安装提示
    deferredPrompt.prompt();
    
    // 等待用户响应
    const { outcome } = await deferredPrompt.userChoice;
    console.log(`用户选择: ${outcome}`);
    
    // 无论结果如何，都清除保存的事件
    deferredPrompt = null;
    
    // 隐藏安装按钮
    hideInstallPromotion();
});

// 监听appinstalled事件，确认应用已安装
window.addEventListener('appinstalled', (evt) => {
    console.log('应用已安装');
    // 隐藏安装按钮
    hideInstallPromotion();
});

2. 安装条件

浏览器通常要求满足以下条件才会触发beforeinstallprompt事件：

  • 应用必须通过HTTPS提供
  • 应用必须有Web App Manifest，包含必要的字段（name、short_name、icons等）
  • 应用必须注册了Service Worker
  • 用户必须与应用有足够的交互（例如，访问过几次网站）
  • 用户尚未安装应用

3. 检测应用是否已安装

我们可以检测应用是否已经以PWA模式运行：

// 检测是否以独立模式运行（已安装）
function isStandalone() {
    return (window.matchMedia('(display-mode: standalone)').matches) ||
           (window.navigator.standalone) ||
           document.referrer.includes('android-app://');
}

// 应用启动时检查
if (isStandalone()) {
    console.log('应用以独立模式运行');
    hideInstallPromotion();
} else {
    console.log('应用在浏览器中运行');
}

五、PWA的性能优化

PWA的一大优势是性能。以下是一些优化PWA性能的策略：

1. 关键渲染路径优化

  • 内联关键CSS：将首屏所需的CSS直接内联到HTML中
  • 延迟加载非关键JavaScript：使用defer或async属性
  • 预连接到重要的第三方域名：使用<link rel="preconnect">
  • 预加载关键资源：使用<link rel="preload">
<!-- 预连接到重要的第三方域名 -->
<link rel="preconnect" href="https://api.example.com">
<link rel="preconnect" href="https://cdn.example.com">

<!-- 预加载关键资源 -->
<link rel="preload" href="/css/critical.css" as="style">
<link rel="preload" href="/js/critical.js" as="script">
<link rel="preload" href="/fonts/main.woff2" as="font" crossorigin>

2. 资源优化

  • 图片优化：使用适当的格式（WebP、AVIF），响应式图片
  • 代码分割：将JavaScript代码分割成更小的块，按需加载
  • 压缩资源：压缩HTML、CSS、JavaScript和图片
  • 使用HTTP/2或HTTP/3：减少连接开销，支持多路复用

3. 离线页面优化

  • Web App Manifest：Chrome 38+、Firefox 44+、Edge 79+、Safari 11.3+（部分支持）

  • Background Sync：Chrome 49+、Firefox 44+（通过ServiceWorkerContainer.sync）、Edge 79+、Safari（不支持）
iOS/Safari注意事项：虽然Safari已经开始支持PWA的核心功能，但在某些方面的实现与其他浏览器有所不同。例如，在iOS上安装的PWA无法使用推送通知（直到iOS 16.4+），并且会受到一些平台限制。

2. 使用Workbox简化Service Worker开发

Workbox是Google的一个库，它提供了一套工具，简化Service Worker的开发。下面是一个使用Workbox的简单示例：

// 安装Workbox
// npm install workbox-cli --save-dev
// 或使用CDN

// service-worker.js - 使用Workbox
importScripts('https://storage.googleapis.com/workbox-cdn/releases/6.5.4/workbox-sw.js');

// 调试模式
workbox.setConfig({ debug: false });

// 预缓存Webpack构建生成的文件
workbox.precaching.precacheAndRoute([
    { url: '/', revision: '1' },
    { url: '/index.html', revision: '1' },
    { url: '/css/style.css', revision: '1' },
    { url: '/js/main.js', revision: '1' },
    { url: '/images/icon-192x192.png', revision: '1' },
    { url: '/offline.html', revision: '1' }
]);

// 对静态资源使用缓存优先策略
workbox.routing.registerRoute(
    ({ request }) => request.destination === 'image' ||
                   request.destination === 'font' ||
                   request.destination === 'style',
    new workbox.strategies.CacheFirst({
        cacheName: 'static-assets',
        plugins: [
            new workbox.expiration.ExpirationPlugin({
                maxEntries: 60,
                maxAgeSeconds: 30 * 24 * 60 * 60, // 30天
            }),
        ],
    })
);

// 对API请求使用网络优先策略
workbox.routing.registerRoute(
    ({ url }) => url.pathname.startsWith('/api/'),
    new workbox.strategies.NetworkFirst({
        cacheName: 'api-cache',
        plugins: [
            new workbox.expiration.ExpirationPlugin({
                maxEntries: 50,
                maxAgeSeconds: 24 * 60 * 60, // 1天
            }),
        ],
    })
);

// 对HTML页面使用快速回退策略
workbox.routing.registerRoute(
    ({ request }) => request.destination === 'document',
    new workbox.strategies.StaleWhileRevalidate({
        cacheName: 'pages-cache',
    })
);

// 离线回退
workbox.routing.setCatchHandler(({ event }) => {
    if (event.request.destination === 'document') {
        return caches.match('/offline.html');
    }
    return Response.error();
});

// 推送通知处理
self.addEventListener('push', (event) => {
    const data = event.data.json();
    const options = {
        body: data.body,
        icon: '/images/icon-192x192.png',
        badge: '/images/badge-72x72.png',
        data: {
            url: data.url || '/'
        }
    };
    
    event.waitUntil(
        self.registration.showNotification(data.title, options)
    );
});

self.addEventListener('notificationclick', (event) => {
    event.notification.close();
    
    event.waitUntil(
        clients.matchAll({type: 'window'}).then(windowClients => {
            // 检查是否已经有打开的窗口
            for (const client of windowClients) {
                if (client.url === event.notification.data.url && 'focus' in client) {
                    return client.focus();
                }
            }
            // 否则打开新窗口
            if (clients.openWindow) {
                return clients.openWindow(event.notification.data.url);
            }
        })
    );
});

  • 更完善的标准和规范

十、总结与最佳实践

渐进式Web应用（PWA）结合了Web和原生应用的优势，为用户提供了更好的体验。通过实施PWA，你可以：

  • 提高应用性能和响应速度
  • 提供离线访问能力
  • 增加用户参与度和留存率
  • 扩大用户基础，包括低网速地区的用户
  • 减少数据使用量，节省用户流量

实施PWA的最佳实践：

  1. 从小处开始：先实现基本的离线功能，然后逐步添加更高级的特性
  2. 关注性能：确保应用快速加载和响应
  3. 优化用户体验：设计良好的安装流程和通知体验
  4. 测试不同设备和网络环境：确保在各种条件下都能正常工作
  5. 持续监控和改进：使用分析工具监控性能和用户行为
  6. 保持内容新鲜：实现有效的缓存更新策略
  7. 遵循标准：确保符合Web标准和最佳实践

PWA代表了Web应用的未来发展方向，通过采用这些技术和最佳实践，你可以创建更现代、更强大、更用户友好的Web应用，为用户提供卓越的体验，同时也为你的业务带来显著的价值。
