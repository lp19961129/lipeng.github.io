Vue 3组合式API指南

2023-09-12 阅读时间: 18分钟 作者: 技术博主 分类: Vue

Vue 3引入了全新的组合式API (Composition API)，它提供了一种更灵活、更强大的方式来组织和复用组件逻辑。本文将详细介绍Vue 3组合式API的核心概念、使用方法和最佳实践。

一、组合式API概述

1. 什么是组合式API？

组合式API是Vue 3中引入的一套新的API，它允许我们根据逻辑关注点来组织代码，而不是强制按照选项类型来组织。这使得我们可以更灵活地复用逻辑，更好地处理复杂组件，并提供了更好的TypeScript支持。

2. 为什么需要组合式API？

在Vue 2中，使用选项式API (Options API)组织代码时，随着组件变得复杂，逻辑关注点会散落在组件的不同部分，使得代码难以理解和维护。组合式API解决了这个问题，它允许我们将相关的逻辑组织在一起，并使逻辑复用变得更加简单。

注意

组合式API是Vue 3的新特性，但Vue 3仍然完全支持选项式API。你可以根据项目需求和团队偏好选择使用哪种API风格。

二、基础API

1. setup函数

setup函数是组合式API的入口点，它在组件实例创建之前执行，用于设置组件的响应式状态、计算属性、方法等。

setup函数基本用法
<script>
export default {
  // 在组件实例创建前执行
  setup() {
    console.log('组件实例创建前执行');
    
    // 定义响应式状态
    const count = 0;
    
    // 定义方法
    function increment() {
      console.log('递增');
    }
    
    // 返回需要在模板中使用的内容
    return {
      count,
      increment
    };
  }
};
</script>

<template>
  <div>
    <p>Count: {{ count }}</p>
    <button @click="increment">Increment</button>
  </div>
</template>

setup函数的参数

setup函数接收两个参数：

  • props：一个响应式对象，包含组件接收的所有props
  • context：一个上下文对象，包含attrs、slots、emit等属性
setup函数参数
<script>
export default {
  props: {
    title: {
      type: String,
      default: 'Default Title'
    }
  },
  emits: ['update-title'],
  setup(props, context) {
    // props是响应式的，但不能直接修改
    console.log('Title prop:', props.title);
    
    // 访问context属性
    const { attrs, slots, emit } = context;
    
    // 触发自定义事件
    function updateTitle(newTitle) {
      emit('update-title', newTitle);
    }
    
    return {
      updateTitle
    };
  }
};
</script>
警告: 虽然props是响应式的，但你不能在setup函数中直接修改props的值，应该通过emit事件通知父组件更新。

2. 响应式状态

Vue 3提供了几种创建响应式状态的方法：ref和reactive。

ref

ref用于创建一个响应式的引用对象，它可以包含任何类型的值。对于基本类型值，应该使用ref。

ref基本用法
<script>
import { ref } from 'vue';

export default {
  setup() {
    // 创建响应式状态
    const count = ref(0);
    const message = ref('Hello Vue 3');
    
    // 访问和修改ref的值
    function increment() {
      // 需要通过.value访问和修改ref的值
      count.value++;
    }
    
    function updateMessage() {
      message.value = 'Updated message';
    }
    
    return {
      count,
      message,
      increment,
      updateMessage
    };
  }
};
</script>

<template>
  <div>
    <p>Count: {{ count }}</p>
    <p>Message: {{ message }}</p>
    <button @click="increment">Increment</button>
    <button @click="updateMessage">Update Message</button>
  </div>
</template>
提示: 在模板中使用ref时，Vue会自动解包，不需要使用.value。但在JavaScript代码中，必须通过.value访问和修改ref的值。

reactive

reactive用于创建一个响应式的对象，适用于包含多个属性的复杂对象。

reactive基本用法
<script>
import { reactive } from 'vue';

export default {
  setup() {
    // 创建响应式对象
    const user = reactive({
      name: '张三',
      age: 25,
      address: {
        city: '北京',
        district: '朝阳区'
      }
    });
    
    // 直接修改reactive对象的属性
    function updateUser() {
      user.name = '李四';
      user.age = 26;
      user.address.city = '上海';
    }
    
    return {
      user,
      updateUser
    };
  }
};
</script>

<template>
  <div>
    <h2>用户信息</h2>
    <p>姓名: {{ user.name }}</p>
    <p>年龄: {{ user.age }}</p>
    <p>地址: {{ user.address.city }}, {{ user.address.district }}</p>
    <button @click="updateUser">更新用户信息</button>
  </div>
</template>

ref vs reactive

选择使用ref还是reactive取决于你的具体需求：

  • 对于基本类型值（字符串、数字、布尔值等），使用ref
  • 对于复杂对象，两者都可以使用，但reactive提供了更自然的访问方式
  • 如果需要将值传递给函数并保持响应性，ref更合适，因为它是一个引用
特性 ref reactive
适用类型 任意类型 仅对象类型
访问值 需要通过.value 直接访问属性
模板中使用 自动解包 直接使用
重新赋值 通过count.value = newValue 不能直接替换整个对象

3. 计算属性和侦听器

computed

computed用于创建一个计算属性，它的值基于其他响应式状态，并会被缓存，只有在依赖变化时才会重新计算。

computed基本用法
<script>
import { ref, computed } from 'vue';

export default {
  setup() {
    const count = ref(0);
    
    // 创建只读计算属性
    const doubleCount = computed(() => count.value * 2);
    
    // 创建可写计算属性
    const fullName = computed({
      get: () => `${firstName.value} ${lastName.value}`,
      set: (newValue) => {
        const parts = newValue.split(' ');
        firstName.value = parts[0];
        lastName.value = parts[1] || '';
      }
    });
    
    const firstName = ref('张');
    const lastName = ref('三');
    
    function increment() {
      count.value++;
    }
    
    return {
      count,
      doubleCount,
      fullName,
      increment
    };
  }
};
</script>

<template>
  <div>
    <p>Count: {{ count }}</p>
    <p>Double Count: {{ doubleCount }}</p>
    <button @click="increment">Increment</button>
    
    <div>
      <p>First Name: {{ firstName }}</p>
      <p>Last Name: {{ lastName }}</p>
      <p>Full Name: {{ fullName }}</p>
      <button @click="fullName = '李四'">Set Full Name</button>
    </div>
  </div>
</template>

watch

watch用于监听响应式状态的变化，并在状态变化时执行副作用。

watch基本用法
<script>
import { ref, reactive, watch } from 'vue';

export default {
  setup() {
    const count = ref(0);
    const message = ref('Hello');
    
    // 监听单个ref
    watch(count, (newCount, oldCount) => {
      console.log(`Count changed from ${oldCount} to ${newCount}`);
    });
    
    // 监听多个源
    watch([count, message], ([newCount, newMessage], [oldCount, oldMessage]) => {
      console.log('Count or message changed');
    });
    
    // 监听reactive对象
    const user = reactive({
      name: '张三',
      age: 25
    });
    
    // 监听整个对象
    watch(user, (newUser, oldUser) => {
      console.log('User changed');
    }, { deep: true }); // 需要设置deep: true来深度监听
    
    // 监听对象的特定属性
    watch(() => user.name, (newName, oldName) => {
      console.log(`Name changed from ${oldName} to ${newName}`);
    });
    
    // 立即执行watch
    watch(count, (newCount) => {
      console.log(`Current count: ${newCount}`);
    }, { immediate: true }); // 设置immediate: true立即执行
    
    function increment() {
      count.value++;
    }
    
    function updateMessage() {
      message.value = 'Updated';
    }
    
    function updateUser() {
      user.name = '李四';
      user.age = 26;
    }
    
    return {
      count,
      message,
      user,
      increment,
      updateMessage,
      updateUser
    };
  }
};
</script>

watchEffect

watchEffect是一种更简洁的监听方式，它会自动追踪依赖，并在依赖变化时重新运行副作用函数。

watchEffect基本用法
<script>
import { ref, watchEffect } from 'vue';

export default {
  setup() {
    const count = ref(0);
    const message = ref('Hello');
    
    // 创建副作用，自动追踪使用到的响应式状态
    const stop = watchEffect(() => {
      console.log(`Count: ${count.value}, Message: ${message.value}`);
      // 可以在组件卸载时停止监听
      return () => {
        console.log('清理副作用');
      };
    });
    
    function increment() {
      count.value++;
    }
    
    function updateMessage() {
      message.value = 'Updated';
    }
    
    function stopWatching() {
      // 手动停止监听
      stop();
    }
    
    return {
      count,
      message,
      increment,
      updateMessage,
      stopWatching
    };
  }
};
</script>

watch vs watchEffect

  • watch需要明确指定要监听的数据源，并且可以访问旧值和新值
  • watchEffect会自动追踪所有使用到的响应式状态，但无法访问旧值
  • watch更适合精确控制监听的数据源
  • watchEffect更简洁，适合简单的副作用场景

4. 生命周期钩子

在组合式API中，生命周期钩子是通过导入相应的函数来使用的，并且只能在setup函数中调用。

生命周期钩子基本用法
<script>
import { 
  onMounted, 
  onUpdated, 
  onUnmounted, 
  onBeforeMount, 
  onBeforeUpdate, 
  onBeforeUnmount, 
  onErrorCaptured, 
  onRenderTracked, 
  onRenderTriggered 
} from 'vue';

export default {
  setup() {
    // 组件挂载前
    onBeforeMount(() => {
      console.log('组件挂载前');
    });
    
    // 组件挂载后
    onMounted(() => {
      console.log('组件挂载后');
      // 在这里可以访问DOM元素
    });
    
    // 组件更新前
    onBeforeUpdate(() => {
      console.log('组件更新前');
    });
    
    // 组件更新后
    onUpdated(() => {
      console.log('组件更新后');
    });
    
    // 组件卸载前
    onBeforeUnmount(() => {
      console.log('组件卸载前');
      // 在这里清理副作用，如移除事件监听器
    });
    
    // 组件卸载后
    onUnmounted(() => {
      console.log('组件卸载后');
    });
    
    // 捕获子组件错误
    onErrorCaptured((err, instance, info) => {
      console.log('错误捕获:', err, info);
      // 返回false可以阻止错误继续传播
      return false;
    });
    
    // 仅开发环境使用 - 跟踪渲染依赖
    onRenderTracked((event) => {
      console.log('渲染跟踪:', event);
    });
    
    // 仅开发环境使用 - 跟踪渲染触发
    onRenderTriggered((event) => {
      console.log('渲染触发:', event);
    });
    
    return {};
  }
};
</script>
选项式API 组合式API
beforeCreate setup函数内部（替代）
created setup函数内部（替代）
beforeMount onBeforeMount
mounted onMounted
beforeUpdate onBeforeUpdate
updated onUpdated
beforeUnmount onBeforeUnmount
unmounted onUnmounted
errorCaptured onErrorCaptured
renderTracked onRenderTracked
renderTriggered onRenderTriggered

三、高级API

1. toRef和toRefs

当我们想从一个reactive对象中提取单个属性并保持其响应性时，可以使用toRef或toRefs。

toRef和toRefs基本用法
<script>
import { reactive, toRef, toRefs } from 'vue';

export default {
  setup() {
    // 创建响应式对象
    const state = reactive({
      count: 0,
      message: 'Hello',
      user: {
        name: '张三',
        age: 25
      }
    });
    
    // 使用toRef提取单个属性
    const countRef = toRef(state, 'count');
    
    // 使用toRefs将整个对象转换为refs
    const stateAsRefs = toRefs(state);
    
    function increment() {
      // 修改原始state中的count
      state.count++;
    }
    
    function updateName() {
      // 通过refs修改
      stateAsRefs.user.value.name = '李四';
    }
    
    return {
      // 直接返回整个state
      state,
      // 返回单个ref
      countRef,
      // 返回转换后的refs
      ...stateAsRefs,
      increment,
      updateName
    };
  }
};
</script>

<template>
  <div>
    <p>Count via state: {{ state.count }}</p>
    <p>Count via toRef: {{ countRef }}</p>
    <p>Count via toRefs: {{ count }}</p>
    <p>Message: {{ message }}</p>
    <p>User Name: {{ user.name }}</p>
    <p>User Age: {{ user.age }}</p>
    
    <button @click="increment">Increment</button>
    <button @click="updateName">Update Name</button>
  </div>
</template>
提示: 使用toRefs可以将reactive对象的所有属性转换为独立的ref，这样在模板中就可以直接使用属性名而不需要通过对象访问，同时还能保持响应性。

2. shallowRef和shallowReactive

Vue 3提供了浅层响应式API，用于创建只监听顶层属性变化的响应式状态。

shallowRef和shallowReactive基本用法
<script>
import { shallowRef, shallowReactive } from 'vue';

export default {
  setup() {
    // 创建浅层ref
    // 只监听.value的变化，不监听对象内部属性的变化
    const shallowCount = shallowRef({
      value: 0
    });
    
    // 创建浅层reactive对象
    // 只监听顶层属性的变化，不监听嵌套对象属性的变化
    const shallowState = shallowReactive({
      count: 0,
      user: {
        name: '张三',
        age: 25
      }
    });
    
    function updateShallowRefValue() {
      // 这不会触发更新，因为我们修改的是嵌套属性
      shallowCount.value.value++;
    }
    
    function replaceShallowRef() {
      // 这会触发更新，因为我们替换了整个.value
      shallowCount.value = { value: shallowCount.value.value + 1 };
    }
    
    function updateTopLevel() {
      // 这会触发更新，因为修改的是顶层属性
      shallowState.count++;
    }
    
    function updateNested() {
      // 这不会触发更新，因为修改的是嵌套对象的属性
      shallowState.user.name = '李四';
    }
    
    return {
      shallowCount,
      shallowState,
      updateShallowRefValue,
      replaceShallowRef,
      updateTopLevel,
      updateNested
    };
  }
};
</script>
注意: 浅层响应式API主要用于性能优化，在大多数情况下应该使用普通的ref和reactive。

3. readonly和shallowReadonly

readonly用于创建一个只读的响应式对象，shallowReadonly用于创建一个只读的浅层响应式对象。

readonly和shallowReadonly基本用法
<script>
import { reactive, readonly, shallowReadonly } from 'vue';

export default {
  setup() {
    // 创建普通响应式对象
    const state = reactive({
      count: 0,
      user: {
        name: '张三',
        age: 25
      }
    });
    
    // 创建深只读对象
    const deepReadonlyState = readonly(state);
    
    // 创建浅只读对象
    const shallowReadonlyState = shallowReadonly(state);
    
    function updateOriginal() {
      // 修改原始对象，所有引用都会更新
      state.count++;
      state.user.name = '李四';
    }
    
    function tryUpdateDeepReadonly() {
      // 这不会生效，因为对象是只读的，而且会在开发环境发出警告
      deepReadonlyState.count = 10;
      deepReadonlyState.user.name = '王五';
    }
    
    function tryUpdateShallowReadonly() {
      // 这不会生效，而且会在开发环境发出警告
      shallowReadonlyState.count = 10;
      
      // 这会生效，因为shallowReadonly只保护顶层属性
      shallowReadonlyState.user.name = '王五';
    }
    
    return {
      state,
      deepReadonlyState,
      shallowReadonlyState,
      updateOriginal,
      tryUpdateDeepReadonly,
      tryUpdateShallowReadonly
    };
  }
};
</script>

4. customRef

customRef用于创建自定义的ref，它允许我们自定义响应式行为。

customRef基本用法
<script>
import { customRef } from 'vue';

export default {
  setup() {
    // 创建防抖ref
    function useDebouncedRef(value, delay = 300) {
      let timeout;
      
      return customRef((track, trigger) => ({
        get() {
          // 追踪依赖
          track();
          return value;
        },
        set(newValue) {
          // 清除之前的定时器
          clearTimeout(timeout);
          
          // 设置新的定时器
          timeout = setTimeout(() => {
            value = newValue;
            // 触发更新
            trigger();
          }, delay);
        }
      }));
    }
    
    // 使用防抖ref
    const searchQuery = useDebouncedRef('');
    
    return {
      searchQuery
    };
  }
};
</script>

<template>
  <div>
    <input 
      v-model="searchQuery" 
      placeholder="搜索..." 
    />
    <p>搜索词: {{ searchQuery }}</p>
  </div>
</template>

5. provide和inject

provide和inject用于在组件树中传递数据，而不需要通过props逐层传递。

provide和inject基本用法
// 父组件
<script>
import { ref, provide } from 'vue';

export default {
  setup() {
    const count = ref(0);
    
    // 提供数据给子组件
    provide('count', count);
    provide('increment', () => {
      count.value++;
    });
    
    return {
      count
    };
  }
};
</script>

// 子组件
<script>
import { inject } from 'vue';

export default {
  setup() {
    // 注入父组件提供的数据
    const count = inject('count', ref(0)); // 第二个参数是默认值
    const increment = inject('increment', () => {});
    
    return {
      count,
      increment
    };
  }
};
</script>

<template>
  <div>
    <p>从父组件接收的count: {{ count }}</p>
    <button @click="increment">增加count</button>
  </div>
</template>
提示: provide和inject的响应式传递：如果提供的值是ref或reactive对象，那么注入后仍然是响应式的。

四、组合式函数

组合式函数（也称为组合式API的自定义Hook）是一种将组合式API逻辑封装为可重用函数的方式。

组合式函数示例
// useCounter.js - 可重用的组合式函数
import { ref, computed } from 'vue';

export function useCounter(initialValue = 0) {
  const count = ref(initialValue);
  const doubleCount = computed(() => count.value * 2);
  
  function increment() {
    count.value++;
  }
  
  function decrement() {
    count.value--;
  }
  
  function reset() {
    count.value = initialValue;
  }
  
  // 返回需要暴露的状态和方法
  return {
    count,
    doubleCount,
    increment,
    decrement,
    reset
  };
}

// 在组件中使用
<script>
import { useCounter } from './useCounter';

export default {
  setup() {
    // 使用组合式函数
    const { count, doubleCount, increment, decrement, reset } = useCounter(10);
    
    return {
      count,
      doubleCount,
      increment,
      decrement,
      reset
    };
  }
};
</script>

<template>
  <div>
    <p>Count: {{ count }}</p>
    <p>Double Count: {{ doubleCount }}</p>
    <button @click="increment">+</button>
    <button @click="decrement">-</button>
    <button @click="reset">Reset</button>
  </div>
</template>

常见的组合式函数示例

useLocalStorage组合式函数
// useLocalStorage.js
import { ref, watchEffect } from 'vue';

export function useLocalStorage(key, initialValue) {
  // 尝试从localStorage读取值
  const storedValue = localStorage.getItem(key);
  const initial = storedValue !== null ? JSON.parse(storedValue) : initialValue;
  
  // 创建ref
  const value = ref(initial);
  
  // 监听值变化，同步到localStorage
  watchEffect(() => {
    localStorage.setItem(key, JSON.stringify(value.value));
  });
  
  return value;
}

// 使用示例
<script>
import { useLocalStorage } from './useLocalStorage';

export default {
  setup() {
    const username = useLocalStorage('username', '');
    const theme = useLocalStorage('theme', 'light');
    
    return {
      username,
      theme
    };
  }
};
</script>
useFetch组合式函数
// useFetch.js
import { ref, onMounted, watch, watchEffect } from 'vue';

export function useFetch(url, options = {}) {
  const data = ref(null);
  const error = ref(null);
  const loading = ref(false);
  
  const fetchData = async () => {
    loading.value = true;
    error.value = null;
    
    try {
      const response = await fetch(url, options);
      if (!response.ok) {
        throw new Error(`HTTP error! status: ${response.status}`);
      }
      data.value = await response.json();
    } catch (err) {
      error.value = err.message;
    } finally {
      loading.value = false;
    }
  };
  
  // 初始加载
  onMounted(fetchData);
  
  // 监听url变化
  watch(() => url, fetchData);
  
  // 返回状态和重新获取方法
  return {
    data,
    error,
    loading,
    refetch: fetchData
  };
}

// 使用示例
<script>
import { ref } from 'vue';
import { useFetch } from './useFetch';

export default {
  setup() {
    const userId = ref(1);
    const { data, loading, error, refetch } = useFetch(
      `https://api.example.com/users/${userId.value}`
    );
    
    return {
      userId,
      data,
      loading,
      error,
      refetch
    };
  }
};
</script>

五、组合式API的最佳实践

1. 逻辑组织

  • 按功能分组代码：将相关的逻辑放在一起，而不是按照API类型组织
  • 使用组合式函数：将可重用的逻辑提取到组合式函数中
  • 遵循关注点分离原则：将UI逻辑和业务逻辑分开

2. 性能优化

  • 使用计算属性缓存计算结果：避免在模板中进行复杂计算
  • 合理使用watch和watchEffect：避免不必要的副作用
  • 使用shallowRef和shallowReactive：对于不需要深层响应的对象
  • 避免不必要的reactive包装：只对需要响应的数据使用响应式API

3. 代码结构

  • 导入语句：按逻辑顺序导入API和组合式函数
  • 状态定义：首先定义所有响应式状态
  • 计算属性和侦听器：然后定义计算属性和侦听器
  • 方法定义：接着定义组件方法
  • 生命周期钩子：最后使用生命周期钩子
  • 返回对象：在setup函数末尾返回需要暴露的内容
推荐的代码结构
<script>
// 1. 导入
import { ref, computed, watch, onMounted } from 'vue';
import { useCounter } from './composables/useCounter';
import { useLocalStorage } from './composables/useLocalStorage';

export default {
  props: {
    // props定义
  },
  emits: ['event-name'],
  setup(props, { emit }) {
    // 2. 状态定义
    const count = ref(0);
    const message = ref('Hello');
    const user = ref(null);
    
    // 使用组合式函数
    const { count: counter, increment } = useCounter(10);
    const savedTheme = useLocalStorage('theme', 'light');
    
    // 3. 计算属性
    const doubleCount = computed(() => count.value * 2);
    const fullMessage = computed(() => `${message.value} Vue 3`);
    
    // 4. 侦听器
    watch(count, (newCount) => {
      console.log(`Count changed to ${newCount}`);
    });
    
    watchEffect(() => {
      // 副作用代码
    });
    
    // 5. 方法定义
    function handleClick() {
      count.value++;
      emit('count-updated', count.value);
    }
    
    async function fetchUserData() {
      // 获取用户数据
    }
    
    // 6. 生命周期钩子
    onMounted(() => {
      fetchUserData();
      console.log('组件已挂载');
    });
    
    // 7. 返回
    return {
      count,
      message,
      doubleCount,
      fullMessage,
      counter,
      savedTheme,
      handleClick,
      increment
    };
  }
};
</script>

六、实际应用场景

1. 表单处理

组合式API可以更灵活地处理表单状态和验证逻辑。

使用组合式API处理表单
<script>
import { reactive, computed } from 'vue';

export default {
  setup() {
    // 表单数据
    const form = reactive({
      name: '',
      email: '',
      password: ''
    });
    
    // 表单验证
    const errors = reactive({
      name: '',
      email: '',
      password: ''
    });
    
    // 验证规则
    const validateName = () => {
      if (!form.name.trim()) {
        errors.name = '姓名不能为空';
        return false;
      }
      errors.name = '';
      return true;
    };
    
    const validateEmail = () => {
      if (!form.email.trim()) {
        errors.email = '邮箱不能为空';
        return false;
      }
      const emailRegex = /^[^\s@]+@[^\s@]+\.[^\s@]+$/;
      if (!emailRegex.test(form.email)) {
        errors.email = '请输入有效的邮箱地址';
        return false;
      }
      errors.email = '';
      return true;
    };
    
    const validatePassword = () => {
      if (!form.password) {
        errors.password = '密码不能为空';
        return false;
      }
      if (form.password.length < 6) {
        errors.password = '密码长度至少为6个字符';
        return false;
      }
      errors.password = '';
      return true;
    };
    
    // 验证整个表单
    const validateForm = () => {
      const isNameValid = validateName();
      const isEmailValid = validateEmail();
      const isPasswordValid = validatePassword();
      return isNameValid && isEmailValid && isPasswordValid;
    };
    
    // 表单是否有效
    const isFormValid = computed(() => {
      return form.name.trim() && form.email.trim() && form.password &&
             !errors.name && !errors.email && !errors.password;
    });
    
    // 表单提交
    const handleSubmit = () => {
      if (!validateForm()) {
        return;
      }
      
      console.log('提交表单:', form);
      // 这里可以发送API请求
    };
    
    return {
      form,
      errors,
      isFormValid,
      validateName,
      validateEmail,
      validatePassword,
      handleSubmit
    };
  }
};
</script>

<template>
  <form @submit.prevent="handleSubmit">
    <div class="form-group">
      <label for="name">姓名</label>
      <input 
        id="name" 
        v-model="form.name" 
        @blur="validateName"
        type="text" 
        placeholder="请输入姓名"
        :class="{ 'error': errors.name }"
      />
      <span v-if="errors.name" class="error-message">{{ errors.name }}
    </div>
    
    <div class="form-group">
      <label for="email">邮箱</label>
      <input 
        id="email" 
        v-model="form.email" 
        @blur="validateEmail"
        type="email" 
        placeholder="请输入邮箱"
        :class="{ 'error': errors.email }"
      />
      <span v-if="errors.email" class="error-message">{{ errors.email }}
    </div>
    
    <div class="form-group">
      <label for="password">密码</label>
      <input 
        id="password" 
        v-model="form.password" 
        @blur="validatePassword"
        type="password" 
        placeholder="请输入密码"
        :class="{ 'error': errors.password }"
      />
      <span v-if="errors.password" class="error-message">{{ errors.password }}
    </div>
    
    <button type="submit" :disabled="!isFormValid">提交</button>
  </form>
</template>

2. 状态管理

对于不使用Vuex的简单应用，可以使用组合式API创建可共享的状态。

使用组合式API进行简单状态管理
// store.js
import { reactive, readonly } from 'vue';

// 私有状态
const state = reactive({
  user: null,
  isLoggedIn: false,
  cart: []
});

// 公开的只读状态
const publicState = readonly(state);

// 状态修改方法
function login(user) {
  state.user = user;
  state.isLoggedIn = true;
}

function logout() {
  state.user = null;
  state.isLoggedIn = false;
}

function addToCart(product) {
  const existingProduct = state.cart.find(p => p.id === product.id);
  if (existingProduct) {
    existingProduct.quantity += 1;
  } else {
    state.cart.push({ ...product, quantity: 1 });
  }
}

function removeFromCart(productId) {
  state.cart = state.cart.filter(p => p.id !== productId);
}

// 导出
const useStore = () => ({
  state: publicState,
  login,
  logout,
  addToCart,
  removeFromCart
});

export { useStore };

// 在组件中使用
<script>
import { useStore } from './store';

export default {
  setup() {
    const { state, addToCart, removeFromCart } = useStore();
    
    return {
      state,
      addToCart,
      removeFromCart
    };
  }
};
</script>

3. 动画和交互

组合式API可以更灵活地处理动画和交互逻辑。

使用组合式API处理动画
<script>
import { ref, onMounted, onUnmounted } from 'vue';

export default {
  setup() {
    const isVisible = ref(false);
    const animationFrameId = ref(null);
    const startTime = ref(null);
    const duration = 1000; // 动画持续时间（毫秒）
    
    // 动画函数
    function animate(timestamp) {
      if (!startTime.value) startTime.value = timestamp;
      const progress = timestamp - startTime.value;
      const easeProgress = easeOutQuad(progress / duration);
      
      // 更新动画属性
      const element = document.getElementById('animated-box');
      if (element) {
        element.style.opacity = isVisible.value ? easeProgress : 1 - easeProgress;
        element.style.transform = `translateY(${(isVisible.value ? 1 - easeProgress : easeProgress) * 20}px)`;
      }
      
      if (progress < duration) {
        animationFrameId.value = requestAnimationFrame(animate);
      }
    }
    
    // 缓动函数
    function easeOutQuad(t) {
      return t * (2 - t);
    }
    
    // 切换显示状态
    function toggleVisibility() {
      isVisible.value = !isVisible.value;
      startTime.value = null;
      
      if (animationFrameId.value) {
        cancelAnimationFrame(animationFrameId.value);
      }
      
      animationFrameId.value = requestAnimationFrame(animate);
    }
    
    // 清理动画
    onUnmounted(() => {
      if (animationFrameId.value) {
        cancelAnimationFrame(animationFrameId.value);
      }
    });
    
    return {
      isVisible,
      toggleVisibility
    };
  }
};
</script>

<template>
  <div>
    <button @click="toggleVisibility">切换动画</button>
    <div 
      id="animated-box" 
      class="box"
    >
      动画元素
    </div>
  </div>
</template>

七、总结

Vue 3的组合式API为我们提供了一种更灵活、更强大的方式来组织和复用组件逻辑。通过使用组合式API，我们可以：

  • 更好地组织代码，按逻辑关注点而不是选项类型
  • 更容易地复用逻辑，通过组合式函数
  • 更好地处理复杂组件，将相关逻辑集中在一起
  • 获得更好的TypeScript支持
  • 更好地控制响应式系统，通过各种高级API
最后建议

无论是使用选项式API还是组合式API，选择最适合你的项目和团队的方式才是最重要的。Vue 3完全支持两种API风格，你甚至可以在同一个项目中混合使用它们。
